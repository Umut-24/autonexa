% ==================== MAIN REPORT ====================
\chapter{Design Description}

\section{Top-Down System Description}
Provide a system-level explanation from requirements to subsystem decomposition, interfaces, and integration strategy.

\subsection{Requirements and Design Drivers}
List functional/non-functional requirements, constraints, and assumptions.

\subsection{System Architecture}
Insert architecture diagram and describe each subsystem and data/control flow.

\subsection{Subsystem Descriptions}
For each subsystem, include purpose, inputs/outputs, algorithms, hardware/software components, and verification approach.

\subsection{Interface Definitions}
Document electrical, mechanical, and software interfaces (signals, protocols, message types, API contracts).

\section{Engineering Details and Compliance}

\subsection{Related Calculations}
Include key sizing, control, timing, power, thermal, accuracy, and tolerance calculations.

\subsection{Circuit Diagrams and Flow Charts}
Insert and explain relevant schematics and process/control flow charts.

\subsection{Relevant Standards}
State applicable standards/regulations and how your design satisfies them.

\subsection{Design Decisions and Trade-offs}
Explain alternatives considered and rationale for final choices.

\chapter{Results and Analyses of Performance Tests}

\section{Test Plan Overview}
Define objectives, metrics, test environment, instrumentation, acceptance criteria.

\section{Test Results}
Present quantitative results in tables/figures with pass/fail against criteria.

\section{Analysis}
Interpret the results, identify limitations, root causes of failures, and improvements.

\section{Validation of Final Requirements}
Map final measured outcomes to original requirements.
